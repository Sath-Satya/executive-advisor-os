How to connect a Snowflake source

Connectors 5 min read Updated: April 1, 2026 Views: 4,218
TL;DR

Create a dedicated Snowflake service account with read-only access, generate an RSA keypair, add it in Meridian under Connectors → Snowflake, and paste your account identifier. Your first dataset will be ready in under 2 minutes.

Before you begin

  • Snowflake account with SYSADMIN or SECURITYADMIN role access (to create the service user)
  • Meridian workspace with Admin or Connector Manager role
  • openssl available locally (for keypair generation)

Steps

  1. 1
    Create a read-only service user in Snowflake

    Run the following SQL in a Snowflake worksheet. Replace YOUR_WAREHOUSE and YOUR_DATABASE with your actual names.

    -- Create role and user
CREATE ROLE meridian_reader;
GRANT USAGE ON WAREHOUSE YOUR_WAREHOUSE TO ROLE meridian_reader;
GRANT USAGE ON DATABASE YOUR_DATABASE TO ROLE meridian_reader;
GRANT SELECT ON ALL TABLES IN SCHEMA YOUR_DATABASE.PUBLIC TO ROLE meridian_reader;

CREATE USER meridian_svc
    LOGIN_NAME = 'meridian_svc'
    DEFAULT_ROLE = meridian_reader
    MUST_CHANGE_PASSWORD = FALSE;

GRANT ROLE meridian_reader TO USER meridian_svc;
  2. 2
    Generate an RSA keypair

    Snowflake connector uses key-pair authentication — more secure than password-based access.

    # Generate private key (keep this secret)
openssl genrsa -out meridian_rsa.pem 2048

# Extract public key
openssl rsa -in meridian_rsa.pem -pubout -out meridian_rsa_pub.pem
  3. 3
    Assign the public key to the Snowflake user

    Copy the content of meridian_rsa_pub.pem (everything between the header and footer lines) and run:

    ALTER USER meridian_svc SET RSA_PUBLIC_KEY='MIIBIjANBgkq...(your public key)...';
  4. 4
    Add the connector in Meridian

    Go to Settings → Connectors → Add connector → Snowflake. Fill in the fields:

    Snowflake connector form screenshot
    • Account identifier: orgname-accountname (find in Snowflake → Admin → Accounts)
    • Username: meridian_svc
    • Private key: paste the contents of meridian_rsa.pem
    • Warehouse: YOUR_WAREHOUSE
  5. 5
    Test the connection

    Click Test connection. You should see a green Connected badge within 10 seconds. If the test fails, see Troubleshooting Snowflake connections.

Best practice: Restrict the meridian_reader role to specific schemas instead of granting access to all tables. This follows the principle of least privilege and reduces your attack surface.
