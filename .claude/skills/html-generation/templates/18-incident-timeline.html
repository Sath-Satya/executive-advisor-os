Root Cause & Contributing Factors

Root Cause

  • PR #4812 changed autoscaler `minReplicas` from 4 → 1 without SRE review gate
  • Deployment landed during off-hours (22:00 UTC); scale-down triggered overnight
  • `scaleDownCooldown` of 2 minutes insufficient to prevent aggressive scale-down

Contributing Factors

  • No autoscaler config change policy requiring SRE sign-off
  • Lag alert threshold too high (2M) — should fire earlier at 500K
  • No canary deployment for infra config changes
  • Runbook for consumer lag mitigation lacked manual scale-out steps
