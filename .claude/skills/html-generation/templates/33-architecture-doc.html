Architecture

Meridian Data Platform

A reference architecture for Meridian's multi-tenant data ingestion, indexing, and query infrastructure.

Last revised: 2026-04-01 Scope: Platform Engineering Status: Approved

Components

Connector Hub

Stateless adapter layer. Each connector runs in an isolated container with credential injection via Vault. Supports batch and streaming modes.

Transform Engine

Apache Spark-based transformation layer. Handles schema normalization, deduplication, and partitioning. Auto-scales to 0 between jobs.

Query API

REST and GraphQL endpoints. Tenant isolation enforced at the API gateway. Rate-limited per workspace. P99 query latency target: < 500 ms.

Control Plane

Manages workspace provisioning, RBAC, API key lifecycle, and usage-based billing metering. Built on Temporal workflows for auditability.

AI Query Layer

Translates natural-language questions to validated SQL via a fine-tuned model. Confidence score returned with every response; low-confidence queries escalate to a human review queue.

Index Builder

Builds columnar indexes (Parquet over S3) and vector indexes (pgvector) in parallel. Incremental index updates supported for streaming datasets.

Data Flow

1
ASYNC

Source extraction

Connector Hub polls or receives webhook events from the source system. Raw payloads pushed to the Event Queue with schema version header. No transformation at this stage.

2
ASYNC

Schema validation & enrichment

Transform Engine consumes events, validates against Schema Registry, enriches with ML tags and embeddings, and writes Parquet partitions to Object Store.

3
ASYNC

Index build

Index Builder reads new Parquet partitions and updates the columnar index and vector store. Dataset status flips to "active" on completion.

4
SYNC

Query execution

Client request hits Query API. Cache is checked first (Redis). On miss, AI Query Layer generates SQL, executor runs against columnar index, results streamed back and cached with 60 s TTL.

5
ASYNC

Observability pipeline

Every event (ingestion, transform, query, error) emits a structured trace to the Observability stack (OpenTelemetry → Grafana Cloud). Tenant-isolated dashboards provisioned automatically on workspace creation.

Design Decisions

Decision Alternative considered Rationale Status
Kafka for event queue AWS SQS / Kinesis Replay capability required for schema migration scenarios; Kafka retention supports 7-day replay. Chosen
Parquet on S3 as primary store Delta Lake / Iceberg Parquet simplicity preferred at current scale. Iceberg migration path documented for > 1 PB threshold. Chosen
pgvector for embeddings Pinecone / Weaviate Reduces vendor surface; co-located with metadata reduces network round trips. Pinecone re-evaluated at 1 B+ vectors. Chosen
Temporal for workflow orchestration Airflow / Step Functions Temporal's code-first approach reduces YAML sprawl; built-in retry semantics suit long-running ingestion jobs. Review at 12 mo
