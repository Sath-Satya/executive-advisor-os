Do / Don’t

Do
Use dark theme (#0a0e14 background) as the default for all developer and enterprise-facing interfaces. Apply rounded corners of 8–12px on cards.
Don’t
Use generic AI purple gradients across the entire UI. Do not apply the purple color to more than one component per page section.
Do
Use DM Sans for all body text and headings. Use DM Mono for code samples, hex values, timestamps, and metric numbers requiring monospaced alignment.
Don’t
Use Arial, Comic Sans, Impact, or Papyrus. Never render the wordmark in a condensed or italicized variant.
Do
Apply hover states as translateY(-1px) combined with a border color shift to the primary blue. Keep transitions under 200ms.
Don’t
Animate background gradients, add particle effects, or apply drop shadows heavier than 0 8px 24px rgba(0,0,0,.3).
