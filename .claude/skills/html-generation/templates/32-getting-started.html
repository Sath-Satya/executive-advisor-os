Quick Start — 10 min

Get started with Meridian

Connect your first data source and run a live query in under 10 minutes. No infrastructure setup required.

1

Install the Meridian SDK

Available for Python 3.9+ and Node.js 18+

bash 
pip install meridian-sdk
Tip: We recommend using a virtual environment (python -m venv .venv) to keep dependencies isolated.
2

Configure your credentials

Authenticate using your workspace API key

Create a .env file in your project root. Never commit this file to version control.

.env 
MERIDIAN_API_KEY=mk_live_xxxxxxxxxxxxxxxxxxxx
MERIDIAN_WORKSPACE_ID=ws_your_workspace_id
MERIDIAN_REGION=us-east-1
Python 
import meridian
from dotenv import load_dotenv

load_dotenv()

# Client auto-reads MERIDIAN_API_KEY and MERIDIAN_WORKSPACE_ID
client = meridian.Client()
print(client.ping())  # {"status": "ok", "latency_ms": 42}
Security note: Add .env to your .gitignore. API keys exposed in git history require rotation.
3

Connect a dataset

Point Meridian at your first data source

Meridian supports 40+ connectors. This example uses a CSV file to get started immediately.

Python 
dataset = client.datasets.create(
    name="Sales Q4 2025",
    source={
        "type": "csv",
        "url": "s3://your-bucket/sales-q4.csv",
    }
)

print(dataset.id)      # ds_01hx4g
print(dataset.status)  # "indexing"
Indexing time: Small files (< 100 MB) are indexed in under 30 seconds. Check dataset.status until it reads "active".
4

Run your first query

Query with natural language or SQL

Python 
result = client.query.run(
    dataset_id=dataset.id,
    q="Top 5 products by revenue last quarter"
)

for row in result.rows:
    print(row)
  • Structured rows you can iterate or export to a DataFrame
  • Auto-generated SQL visible via result.sql
  • Confidence score and reasoning chain via result.explanation
5

Schedule a pipeline

Automate refreshes with a cron expression

Python 
pipeline = client.pipelines.create(
    name="Daily Sales Refresh",
    dataset_id=dataset.id,
    schedule="0 6 * * *",   # every day at 06:00 UTC
    on_failure="notify_slack"
)

print(f"Pipeline {pipeline.id} scheduled")

You are up and running
