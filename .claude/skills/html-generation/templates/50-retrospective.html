Lessons Learned
L-01

Pair programming on complex AI logic before QA reliably surfaces integration edge cases that solo review misses. Expand this practice to all D3-tier deliverables in Phase 3.

L-02

External API documentation from clients should be validated against live endpoints at sprint kickoff, not assumed current. Build a lightweight contract-test step into sprint planning.

L-03

Clear ticket definitions eliminated mid-sprint scope questions entirely. Invest the extra 20 minutes at sprint planning to write acceptance criteria in full sentences, not bullet fragments.

L-04

Architecture decisions that block multiple engineers should have a time-boxed async decision window (24h) before escalating to a synchronous session. This prevents standup bottlenecks.
