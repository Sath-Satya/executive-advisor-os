Mitigation Actions by Risk

R-001 — Critical AI Model Bias / Discrimination

Implement quarterly bias audits using stratified sampling by race/ethnicity, gender, and geography. Add real-time demographic parity monitoring to the AI observability stack. Engage external equity review firm for annual audit.

Due: {{MITIGATION_DATE}} Owner: CTO / CMO Target Score: 12 (High)
R-002 — Critical ePHI Data Breach

Enable encryption at rest on all ePHI data stores. Implement data loss prevention (DLP) controls at API egress points. Conduct quarterly penetration testing of AI inference endpoints. Validate BAA coverage for all sub-processors.

Due: {{MITIGATION_DATE}} Owner: CISO Target Score: 10 (High)
R-003 — High AI Model Drift

Implement automated model performance monitoring with decision distribution alerts. Define and enforce a model revalidation trigger tied to formulary updates, guideline changes, and quarterly drift metrics. Establish a fast-path revalidation SLA of 10 business days.

Due: {{MITIGATION_DATE}} Owner: AI Ops Lead Target Score: 6 (Medium)
R-004 — High Vendor API Downtime

Implement fallback to manual review queue on vendor API timeout (>30s). Negotiate 99.9% uptime SLA with penalty provisions. Deploy a queue-based retry architecture to absorb transient failures without losing requests.

Due: {{MITIGATION_DATE}} Owner: VP Operations Target Score: 8 (Medium)
