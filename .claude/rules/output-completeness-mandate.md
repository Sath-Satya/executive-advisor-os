# Output Completeness Mandate (NON-NEGOTIABLE)

## Purpose
Close the "half-complete deliverable" gap. Markdown source files are NOT final deliverables for executives. Executives consume PPTX, PDF, and HTML — not Marp `.md` files. Any research session that produces `deck_*.md` without the corresponding `.pptx`, or leaves an HTML briefing missing, is **incomplete** regardless of how polished the MD files look.

## Scope
Applies to every Executive Advisor OS research session — `/new-research`, `/continue`, `/recalibrate`.

## Non-Negotiable Rules

### 1. Decks MUST ship in PPTX form
- Every `deck_summary_v{N}.md` MUST be paired with `deck_summary_v{N}.pptx`
- Every `deck_detailed_v{N}.md` MUST be paired with `deck_detailed_v{N}.pptx`
- The Marp MD file is the **source**; the PPTX is the **deliverable**
- Conversion tool: `.claude/scripts/md_to_pptx.py` (python-pptx-based, cream-navy-gold palette)
- Usage: `python .claude/scripts/md_to_pptx.py <input.md> [<output.pptx>]`
- **Automation:** a PostToolUse hook at `.claude/hooks/auto_convert_deck.py` (registered in `.claude/settings.json`) fires after every Write/Edit. When the written file matches `research/*/output/deck_*.md`, the converter runs automatically — Claude does NOT need to remember the conversion command. The hook is advisory (exit 0 always); if python-pptx is missing, it prints the install hint and the `output-verification` skill catches the missing `.pptx` at session end.
- **First-time dependency:** `python-pptx>=1.0.0` is required once per machine. See `SETUP.md` Prerequisites section. Repo `requirements.txt` includes it.

### 2. HTML briefings MUST ship as self-contained files
- Every `briefing_v{N}.html` MUST be offline-safe (no external JS/CSS except Google Fonts)
- The `.html` file is the deliverable; a separate PDF is NOT required unless the exec explicitly requests one

### 3. Requested outputs are binding
- When the exec picks outputs via AskUserQuestion (per `CLAUDE.md` Step 5), that selection becomes a **contract**
- Every requested format must exist in its final user-consumable form before the session claims "complete"
- Executive summary (`executive_summary_v{N}.md`) is generated by default even if not explicitly requested (per CLAUDE.md)

### 4. No "the MD is the deck" shortcuts
- Marp slide outlines are intermediate artifacts. A session that produces only `deck_*.md` files is NOT complete even if the MD is perfectly structured.
- The final deliverable for a deck is a PPTX (the exec may further convert to PDF or Google Slides themselves — they have the native file)

### 5. Completion claim requires verification
- Before saying "complete", "ready", "done", "shipped" for a research session, the `output-verification` skill (or its embedded checklist) MUST pass
- If verification fails, the session is NOT complete — regenerate the missing artifact before claiming

## Mandatory file inventory per output type

| Requested output | Files that MUST exist before "complete" |
|---|---|
| Executive summary | `executive_summary_v{N}.md` |
| Detailed analysis | `detailed_analysis_v{N}.md` (+ `.plan.md` if >300 lines) |
| HTML briefing | `briefing_v{N}.html` (+ `briefing.plan.md` if >300 lines) |
| Deck — summary (≤3 slides) | `deck_summary_v{N}.md` AND `deck_summary_v{N}.pptx` |
| Deck — detailed (10–12 slides) | `deck_detailed_v{N}.md` AND `deck_detailed_v{N}.pptx` AND `deck_detailed.plan.md` AND per-slide MDs in `slides/` |

## How to Apply

When the session approaches completion:
1. Enumerate the output selection captured in memory (Round 5-ish AskUserQuestion)
2. Run the mandatory file inventory above against the `output/` folder
3. For any missing artifact — regenerate before claiming complete
4. For any deck where MD exists but PPTX is missing — run `md_to_pptx.py` immediately
5. Update the dashboard ONLY after all mandatory files are on disk

## Enforcement

- `output-verification` skill — pre-completion audit; blocks "done" claims until mandatory files exist
- Dashboard regeneration rule — file tree displays the full set; missing files appear with `missing` class (strikethrough) which is an immediate visual flag
- Session handover — must list every output file actually on disk, not what "should have been" generated
- Follow-up review cadence: if a session ever closes without a PPTX, flag it as a Principle 7 learning and add a `_learnings.md` entry in `output-verification`

## Related rules

- **`.claude/rules/exec-friendly-clarification.md`** — governs the *language* of clarification questions (business phrasing, not methodology jargon). This completeness mandate governs the *artifacts* at the end. Both rules emerged from the same first-research review (`cms-0057-payer-implementation`, 2026-04-21) and are designed to work together: exec-friendly language during the interview, complete deliverables at the end.
- **`.claude/rules/large-artifact-generation.md`** — plan-in-MD-first discipline for any output crossing 300 lines or 40KB. Complements this rule by preventing truncated artifacts.

## Why This Rule Exists

Precedent: 2026-04-21 session on `cms-0057-payer-implementation` produced 5 MD decks + HTML briefing but failed to convert Marp source to PPTX. Exec flagged the gap — Marp MD is not a presentable deliverable for a payer leadership team. The executive's workflow expects `.pptx` files that open in PowerPoint; the intermediate Marp source is of no use without conversion. Root cause: no mandatory verification step before claiming completion. This rule and the `output-verification` skill are the fix.
